# Portafolio de Análisis y Diseño de Sistemas

Este repositorio contiene una colección de ejercicios prácticos de análisis y diseño de sistemas de software.
El objetivo es demostrar la capacidad de relevar requerimientos, modelar procesos de negocio y diseñar arquitecturas lógicas utilizando el estándar UML.

## Propósito del portafolio

Como estudiante de ingeniería informática y futuro desarrollador, considero que un buen software se construye a partir de un análisis sólido.

En este repositorio se aplican los siguientes conceptos:

* Ingeniería de requerimientos: especificación de casos de uso y reglas de negocio.
* Modelado estático: diagramas de clases de análisis (Boundary, Control, Entity) y modelos de dominio.
* Modelado dinámico: diagramas de comunicación, actividades y estados.
* Diseño de interfaces: prototipos iniciales (mockups).

## Casos de estudio destacados

A continuación se presentan algunos de los casos más relevantes incluidos en el repositorio:

* Air Patagonia: logística, gestión de paquetes y restricciones de tiempo.
* LIME: movilidad urbana, integración con IoT (GPS/3G) y manejo de estados.
* Service Station: fidelización de clientes, automatización y gestión de inventario.
* LoDePelusa: sistema cashless, control de accesos y seguridad.
* Self-Checkout: entorno retail, validación por hardware (peso) y procesamiento de pagos.

Cada caso incluye documentación completa con los distintos artefactos de análisis.

## Herramientas y metodologías

* Lenguaje de modelado: UML
* Herramientas de modelado: Draw.io, Visual Paradigm, Balsamiq Wirefirmes.
* Enfoque: análisis orientado a objetos con separación de responsabilidades

## Estructura del repositorio

Cada carpeta corresponde a un caso de estudio independiente e incluye:

* Diagrama de casos de uso
* Modelo de dominio
* Especificación de casos de uso
* Interfaz tentativa
* Diagrama de clases de análisis
* Diagrama de comunicación
* Diagramas de comportamiento (actividades, estados, según corresponda)
